The fact that my name was removed in the later version was especially frustrating for me because core ideas from the paper came from the optimization work that I had been doing as part of the team working on a project we called the Deep Learning Accelerator (DLA). DLA is a design that can be loaded onto FPGAs so that the FPGA can be used to accelerate AI models. My job was to try to improve the performance of the design and add support for new neural networks. In marketing materials, Intel refers to DLA as the FPGA AI Suite.

One of the things I did when I was optimizing the design was a technique that involved factoring out the exponent from a group of floating point numbers that you were intending to multiply, so that the numbers can by multiplied by fixed-point multipliers rather than floating-point multipliers, with the result then accumulating in a floating-point accumulator. On an FPGA, this roughly doubles the number of multipliers that you can use, and therefore doubles the speed. I was very pleased with this optimization. After I implemented it, I Googled and realized that there was a technique called block floating point that had been used in other contexts that was similar. After I found out, I started using the same phrase to refer to my work so that people would understand what I had implemented.

It appears that the block floating point technique has been useful for Intel customers. From what I can tell, it appears that the technique has been useful enough that Intel modified the design of their FPGAs so that they can perform block floating point even more efficiently. According to the documentation I am able to find online, it appears that starting with the Agilex 5 and the Stratix 10 NX, Intel included a modification to the DSP block on the FPGA that allows multipliers configured for fixed-point to be directly connected to a floating-point accumulator, with a shared exponent passed into the accumulator. It is more efficient to directly connect them with a dedicated circuit rather than to use normal reconfigurable logic to connect them, like I did for the DLA project. If Intel made this change, it is likely because their customers are actually using it, and so they are probably making a lot of money from my idea.

The documentation for the Agilex and Stratix chips appear to include the feature as part of what is called the AI Tensor Block. I found this document[1], which includes a diagram showing a common exponent passed to an accumulator in Figure 6. I also found this paper[2], which unfortunately is not available to the public, however you can see a reference to block floating point in the Abstract. After doing some Googling, I was able to find these slides[3], which very clearly show on Slide 6 (AI Tensor Block Architecture), that the floating-point accumulator is able to accept a shared exponent.

I think these modifications to the FPGA make sense. I just want to prove that the idea I had for the DLA project was indeed useful and valuable. The fact that the idea was a good one helps explain why someone might have been motivated to remove my name from the Intel version of the paper.
